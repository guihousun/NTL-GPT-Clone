# NTL-GPT 近期功能与修复总结（简版）

日期：2026-02-27

## 一、核心功能升级

### 1) UI 与交互
- 统一视觉风格（Nasa全球夜间灯光背景、卡片、面板样式）。
- 推理区流式刷新优化，显著降低闪烁与抖动。
- 增强 Reasoning Graph 与子代理状态卡片（Pending / Running / Done）。
- 支持中英文切换。
- 支持图片等多模态数据上传。

### 2) 用户与会话管理
- 支持按用户名查看、加载、删除历史线程。
- 文件与聊天记录按 `Username + Thread ID` 严格隔离。
- 工作区目录标准化：`inputs/`、`outputs/`、`memory/`。

### 3) 架构升级
- 迁移到 LangChain Deep Agents 框架。
- 多子代理职责更清晰（Engineer / KB / Data Searcher / Code Assistant）。
- 后端路由能力增强（`/inputs`、`/outputs`、`/memories`、`/shared`）。
- 支持 `Skills` 。

### 4) 功能升级
- 支持图片等多模态数据上传（粘贴/选择），并进入统一对话处理链路。

## 二、重点问题修复

### 1) Skills 加载问题
- 当前支持按目录递归发现所有 `SKILL.md` 文件。

### 2) 共享数据安全与可用性
- 支持 `/shared/...` 作为输入读取。
- 增加只读保护：禁止写入 `/shared/...`，所有输出必须写入当前线程 `outputs/`。

### 3) 代码执行链路稳定性
- 修复并增强脚本执行前后流程：保存 -> 读取 -> 执行 ->（必要时）一次轻修复。
- 对 `ScriptNotFoundError` 返回更完整诊断信息（可用脚本列表、恢复建议）。

### 4) 工具与渲染一致性
- `write_todos` 状态统一为 `completed` 语义并修复统计展示。
- 表格类文件读取能力增强（如 `.csv`、`.xlsx`）。

## 三、当前基线规则
- `/shared/...`：可读不可写。
- 所有生成产物：仅写当前线程 `outputs/`。
- 线程隔离：按 `thread_id` 严格生效。
- Skills：应可被发现并注入（新线程验证）。
