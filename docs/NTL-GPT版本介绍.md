# NTL-GPT版本介绍

最后更新：2026-02-25

## 1. 项目定位
NTL-GPT 是一个面向夜间灯光遥感（NTL）分析的多智能体系统。目标是把“数据检索 -> 边界确认 -> 脚本生成 -> 执行验证 -> 结果输出”整合为可追踪、可复现、可协作的一体化流程。

核心应用方向：
- 灾害影响评估（地震、洪涝、火灾、冲突等）
- 城市发展与经济活动监测（ANTL、区域统计、时序对比）
- 论文方法复现与流程沉淀（Solution/Code/Literature 三类知识）

## 2. 当前系统能力（当前版本）

### 2.1 多智能体协同
- `NTL_Engineer`：任务分解、路由调度、最终收敛与结果整合
- `Data_Searcher`：数据源检索、空间/时间边界确认、数据准备
- `Code_Assistant`：脚本读取、测试链验证、执行与产物输出

### 2.2 知识与检索（RAG）
- 支持 `Solution_RAG`、`Code_RAG`、`Literature_RAG`
- 支持 workflow/theory/code/mixed 等输出模式
- 支持结构化 JSON 合约输出，便于 UI 渲染与后续自动执行
- 文献检索支持复现导向信息提取（方法、参数、时窗、约束）

### 2.3 执行与工程约束
- 线程隔离工作区：`user_data/<thread_id>/{inputs,outputs}`
- 路径安全策略：限制跨线程输出，异常时可恢复到当前线程
- Code 执行链路：先保存脚本，再读取脚本，再执行/验证
- 失败策略：首轮失败进入验证链，有限次修复后再交回 Engineer

### 2.4 前端交互与可观测性（Streamlit）
- 聊天式任务提交 + 推理过程展示
- Reasoning Graph（主路径/工具节点可视化）
- Map View（图层叠加、样式配置、像元识别）
- Result Preview（CSV/JSON/图像等结果预览）
- 运行中状态隔离：常见面板操作不应打断当前任务（除 Stop/New/运行错误）

### 2.5 地理与时序分析能力
- GEE 数据路由与脚本蓝图工具
- 栅格统计、边界裁剪、多尺度缓冲区分析
- 日尺度夜光场景下“首夜”规则约束（如 D+1 逻辑）
- 官方 Daily NTL 快速实验流（独立 experiments 工作区）

## 3. 当前版本的行为边界
- Data_Searcher 负责检索与数据准备，不直接给最终业务结论
- Code_Assistant 以执行与验证为主，不负责业务层最终裁决
- Engineer 负责最终解释、回退决策与跨代理协调
- 输出优先保证可审计（脚本、参数、产物、时间成本）

## 4. 历次更新总览（阶段版）

### 阶段A：知识库建设与路由稳态（2026-02-17）
- 建立 Literature_RAG 入库能力与 metadata 结构
- KB/Searcher 路由修复，减少无效返回与格式不兼容
- Searcher 模型统一至 `qwen3.5-plus`（相关链路）
- 增加反硬编码与泛化优先策略（同类场景可迁移）

### 阶段B：工作流复现增强（2026-02-18）
- 地震 Q20 workflow 扩展为 25/50/100km 多尺度分析
- 固化“首夜过境”时序规则，降低灾害时窗误判
- Solution_RAG 细节优先保留，避免被通用模板覆盖
- Daily NTL 快速实验流与监控能力上线（独立实验域）

### 阶段C：执行链与UI可靠性提升（2026-02-19 ~ 2026-02-24）
- 多轮工具调用流程降噪，handoff 语义稳定
- 推理图谱结构、节点语义、展示逻辑持续优化
- 线程级输出隔离与路径治理增强
- 上传文件理解、粘贴上传、结果预览等交互完善
- 用户隔离策略加强（避免默认共享历史）

### 阶段D：流式运行隔离与体验优化（2026-02-25）
- 聊天流式运行与页面重绘解耦（后台运行控制）
- Stop 行为改进：可立即释放前端运行锁
- 右侧面板防闪优化，运行提示位置优化
- MapView 默认中国视角策略 + 全球误缩放防护
- Engineer 先落盘脚本、Code_Assistant 先读后执行流程固化

## 5. 当前已知风险与后续重点
- 部分历史文档存在早期编码/风格遗留，需持续清理
- 某些外部依赖版本变更会影响测试收集（需固定版本策略）
- 需持续压缩 prompt 体积与工具调用数，降低时延
