# NTL-GPT 近期功能与修复总结（2026-02-27）

本文档汇总近期已完成的核心功能升级与关键问题修复，便于版本评审、回归测试与后续迭代。

## 1. UI 美化与交互优化

### 1.1 视觉风格统一
- 完成整体视觉风格统一：背景图、主面板、侧栏、卡片样式与边框语言一致化。
- 推理区、结果区、输入区的排版结构做了统一收敛，减少割裂感。

### 1.2 流式更新优化（减少闪烁）
- 将高频文本更新与低频图谱更新进行分离，避免同一刷新通道导致的明显闪屏。
- 优化局部刷新策略，减少全页重绘带来的视觉抖动。
- 调整运行提示与聊天区域布局，降低任务执行过程中的 UI 跳动。

### 1.3 推理可视化增强
- 新增/增强 Reasoning Graph 渲染能力。
- 增加子代理生命周期状态卡片（Knowledge Base / Data Searcher / Code Assistant / NTL Engineer），实时显示 Pending/Running/Done。
- 生命周期卡片做了紧凑化处理（高度、间距、信息密度）。

### 1.4 工具输出可读性改进
- `write_todos` 输出支持结构化渲染（可折叠任务列表）。
- ToDo 状态归一化：`done/completed/finished` 等统一到 `completed`，统计与标签一致。

### 1.4.1 多模态上传支持
- 支持图片等多模态数据上传（粘贴/选择），并纳入统一的会话处理链路。

### 1.5 多语言支持
- 支持中英文界面切换（UI 文案按语言状态动态渲染）。
- 关键运行提示、面板标题与状态信息已纳入双语显示逻辑。

---

## 2. 用户管理与会话隔离完善

### 2.1 历史线程管理
- 支持按用户名查看历史线程。
- 支持加载历史对话继续执行。
- 支持删除历史线程记录。

### 2.2 严格隔离（Username + Thread ID）
- 会话级数据与文件目录均按 `thread_id` 隔离。
- 线程级工作区目录规范：
  - `user_data/<thread_id>/inputs`
  - `user_data/<thread_id>/outputs`
  - `user_data/<thread_id>/memory`
- 聊天记录与产物绑定到对应线程，避免跨用户/跨线程串扰。

### 2.3 Thread ID 绑定稳定性
- 关键工具执行链路（下载、脚本执行、文件读写）统一绑定当前线程上下文。
- 降低“文件在别的线程存在、当前线程找不到”的误用概率。

---

## 3. 框架升级：迁移到 LangChain Deep Agents

### 3.1 架构升级
- 从原有多代理编排方案迁移到 Deep Agents 框架。
- 采用 Supervisor + Subagents 模式，职责边界更清晰：
  - `NTL_Engineer`（调度与决策）
  - `Knowledge_Base_Searcher`（方法与任务判级）
  - `Data_Searcher`（数据检索与覆盖校验）
  - `Code_Assistant`（代码执行与修复闭环）

### 3.2 后端能力增强
- 接入 Deep Agents backend（FilesystemBackend + CompositeBackend 路由）。
- `/inputs`、`/outputs`、`/memories`、`/shared` 等虚拟路径与物理存储映射打通。

### 3.3 模型与执行稳定性提升
- 子代理模型可独立配置。
- 提示词与技能（skills）联动加强，减少无效工具调用与重复回路。

## 4. 近期关键 Bug 修复与工程增强

### 4.1 Skills 加载失败（`skills_metadata=[]`）
- 修复 Deep Agents skills source 配置问题：统一使用正确 source 目录。
- 补齐缺失 `SKILL.md` 与 frontmatter。
- 增加启动期 skills 发现校验，避免静默失败。

### 4.2 共享数据可用性与安全性
- 明确支持读取共享数据：`/shared/...` 可作为输入。
- 增加只读保护：禁止将输出写入 `/shared/...`（防止覆盖共享基线数据）。
- 规则：共享可读不可写，所有生成产物必须落在当前线程 `outputs/`。

### 4.3 脚本执行链路稳健性
- `execute_geospatial_script_tool` 在脚本不存在时返回更完整诊断信息：
  - `available_scripts`
  - `last_saved_script_name`
  - `recovery_suggestion`
- 强化“先保存再执行”策略，减少 `ScriptNotFoundError` 重复发生。

### 4.4 Preflight 策略调整
- Preflight 从“默认强阻断”改为“默认辅助与告警优先”，仅在显式开启时阻断。
- 降低误拦截导致的正常任务失败率。

### 4.5 数据与工具行为修复
- `read_workspace_file_tool` 增强对 `.csv/.xlsx` 等表格读取支持。
- `NTL_raster_statistics` 输出路径与线程工作区规范对齐。
- Shapefile 文件组在 UI 中显示逻辑优化（减少冗余噪音）。

### 4.6 代理协同与防循环增强
- 强化 handoff guard，抑制不可用 transfer/handoff 工具幻觉调用。
- 子代理失败时结构化上报，避免无界重试。

---

## 5. 当前行为基线（建议测试时对照）

- 共享数据：可读不可写。
- 线程隔离：所有输入/输出/记忆均按 `thread_id` 隔离。
- 执行顺序：保存脚本 -> 读取脚本 -> 执行脚本 ->（必要时）一次轻修复。
- Skills：应能被发现并注入，不应长期出现 `skills_metadata=[]`（旧线程缓存除外）。
